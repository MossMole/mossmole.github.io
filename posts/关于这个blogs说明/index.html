遇到问题：

  • github发送给QQ邮箱的验证码无法接收到，且排除黑名单，垃圾邮件等邮箱本身的问题。

    原因： 开了服务器代理，有网络延迟，过了5，6分钟后收到邮件，而验证码有效期仅有 1min

    解决方案：换成内陆的代理，收到验证码，登录后立马启用github的备用邮箱功能,追加了gmail邮箱地址，以后就不用担心收不到验证码了。

  • 本地sourcetree和git无法正常访问github,总是报443。

    原因：还是开了代理，当时git没有启用该功能

    解决方案：

    # 查看当前代理是否配置，http.proxy，https.proxy
git config -l

# 有的话，取消代理，提示warn,multi values,使用--unset-all
git config --global --unset http.proxy
git config --global --unset-all https.proxy

#配置目前使用的新代理，以自己使用的为准，用的是clash，7899
git config --global http.proxy 127.0.0.1:7899
git config --global https.proxy 127.0.0.1:7899
